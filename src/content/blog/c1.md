### 3.3 形式参数与实际参数
形式参数与实际参数的区别是，形式参数是在函数定义时声明的参数，而实际参数是在函数调用时传递的参数。

这个道理其实很简单，因为C是编译型语言，因此在编译出的可执行文件中其实只有int main的部分是执行的，只有在调用函数的时候，才会执行函数体中的代码，而实际参数是在函数调用时传递的参数。

**For example！**

这里给出一个例子，来自于Linux C一站式编程的课后作业：

定义一个函数increment，它的作用是把传进来的参数加1。例如：

```
void increment(int x)
{
	x = x + 1;
}

int main(void)
{
	int i = 1, j = 2;
	increment(i); /* i now becomes 2 */
	increment(j); /* j now becomes 3 */
	return 0;
}
```

提问：这个函数能做到增加i和j的值吗？

显然是不能的。

为何？

因为C语言的函数参数传递方式是，按值传递。

也就是，你调用increment（i），程序会申请一块内存给x，把i的值赋值给x，然后执行函数体中的代码。

你不return的话，函数值会被销毁，所以i的值不会改变。


这就是形参和实参的区别了。

永远记住：**形参相当于函数中定义的变量，调用函数传递参数的过程相当于定义形参变量并且用实参的值来初始化。**

结合刚才的话，你应该能理解我在说什么。

如果在一个程序中调用了printf函数却不包含头文件，例如int main(void) { printf("\n"); }，编译时会报警告：warning: incompatible implicit declaration of built-in function ‘printf’，这是为什么呢？

其实，这是因为printf函数是一个库函数，需要在编译时链接到库中，就像后面我们会知道，-lm的重要性。



### 3.4 局部变量与全局变量
我其实对那本书有点好感的，但是那本书里唯一比较败笔的地方就是先介绍形参实参再介绍全局变量和局部变量。

我们把函数中定义的变量称为局部变量（Local Variable），由于形参相当于函数中定义的变量，所以形参也是一种局部变量。在这里“局部”有两层含义：

* 一个函数中定义的变量不能被另一个函数使用。例如print_time中的hour和minute在main函数中没有定义，不能使用，同样main函数中的局部变量也不能被print_time函数使用。

* 每次调用函数时局部变量都表示不同的存储空间。局部变量在每次函数调用时分配存储空间，在每次函数返回时释放存储空间，例如调用print_time(23, 59)时分配hour和minute两个变量的存储空间，在里面分别存上23和59，函数返回时释放它们的存储空间，下次再调用print_time(12, 20)时又分配hour和minute的存储空间，在里面分别存上12和20。

这样就非常清晰，你就会发现哦原来函数每次调用的时候会申请一个新的内存空间，返回的时候释放，这当然和编译有关系，C语言是编译语言，他是先编译后直接像transformer一样执行整个文件的每一个语句，而不是像Python那样一行一行解释执行，所以会有同时运行的需求，自然而然地函数就需要独立变量。

### 什么是结构体？
结构体就是我们自己定义的变量类型，比如说复数。

```
struct complex
{
	double real;
	double imag;
};
```

这是最初级的结构体，这只定义了一个类型，其他什么都没有做。

```
struct complex{
	double real;
	double imag;
}z;
```

上述代码中，我们就像声明了一个变量一样，不过我们先用结构体语言定义了一个结构体，然后声明z是这个类型。

当然了我们还有更方便的方法：

```
struct complex{
	double real;
	double imag;
};

struct complex z;

```

那么，我们要如何访问结构体中的变量呢？

```
z.real=;
z.imag=;
```

当然我们也可以：

```
struct dicarl{
	int x;
	int y;
};

int x=1;
z.x=x;
```

这个结构体其实比较特殊，不过如果你理解了函数是值传递就很好理解了，结构体里面这个参数也是形参，是我们用的时候才会创建的，而且放在z的拿一块内存，他的名字是z.x，和我们的全局变量x是不一样的。

结构体的一个重要思想就是抽象，我们在一生一芯计划中也介绍过这个概念，这几乎是计算机的核心，抽象把复杂系统的许多小问题变成简单系统的单一问题。

* 实现一个用分子分母的格式来表示有理数的结构体rational以及相关的函数，rational结构体之间可以做加减乘除运算，运算的结果仍然是rational。


```
#include<stdio.h>
#include<math.h>

struct rational{
	int a;
	int b;
};

int gcd(int n;int d){
	if (n % d ==0){
		return d;
	}
	return gcd(d,n%d)
}

struct rational make_rational(int a,int b){
	if (b == 0){
		printf("You make a very stupid mistake);
	}
	struct rational ration;
	ration.a=a/gcd(a,b);
	ration.b=b/gcd(a,b);
	return ration;

}

struct rational add_rational(struct rational a,struct rational b)
{
    struct rational answer;
    answer.a=a.a*b.b+b.a*a.b;
    answer.b=a.a*b.b;
    return answer;
}
struct rational sub_rational(struct rational a,struct rational b)
{
    struct rational answer;
    answer.a=a.a*b.b-b.a*a.b;
    answer.b=a.a*b.b;
    return answer;
}
struct rational mul_rational(struct rational a,struct rational b)
{
    struct rational answer;
    answer.a=a.a*b.a;
    answer.b=a.a*b.b;
    return answer;
}
struct rational div_rational(struct rational a,struct rational b)
{
    struct rational answer;
    answer.a=a.a*b.b;
    answer.b=a.b*b.a;
    return answer;
}
void print_rational(struct rational r)
{
    printf("%d/%d\n",r.a,r.b);
}
int main(void)
{
	struct rational a = make_rational(1, 8); /* a=1/8 */
	struct rational b = make_rational(-1, 8); /* b=-1/8 */
	print_rational(add_rational(a, b));
	print_rational(sub_rational(a, b));
	print_rational(mul_rational(a, b));
	print_rational(div_rational(a, b));

	return 0;
}
```

* 习题1：本节只给出了make_from_real_img和make_from_mag_ang函数的实现，请读者自己实现real_part、img_part、magnitude、angle这些函数。

```
#include <math.h>

double real_part(struct complex_struct z)
{
    if (z.t == RECTANGULAR)
        return z.a;
    else
        return z.a * cos(z.b);  // r cosθ
}
double img_part(struct complex_struct z)
{
    if (z.t == RECTANGULAR)
        return z.b;
    else
        return z.a * sin(z.b);  // r sinθ
}
double magnitude(struct complex_struct z)
{
    if (z.t == POLAR)
        return z.a;
    else
        return sqrt(z.a * z.a + z.b * z.b);
}
double angle(struct complex_struct z)
{
    if (z.t == POLAR)
        return z.b;
    else
        return atan2(z.b, z.a);
}
```
